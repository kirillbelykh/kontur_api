# Kontur API

Проект автоматизирует работу с заказами кодов маркировки в Контур.Маркировке:

- авторизация и обновление cookies через Selenium + Яндекс.Браузер;
- создание и отправка заказов КМ;
- выгрузка файлов кодов;
- GUI-приложение на customtkinter.

## Требования

- Windows 10/11
- Интернет
- Доступ к GitHub

## Установка на Windows (рекомендуется)

1. Откройте папку проекта.
2. Запустите `setup.bat` от имени обычного пользователя.
3. Установщик:
   - проверит/установит `git` и `uv` через `winget`;
   - установит Python и зависимости через `uv sync`;
   - проверит Яндекс.Браузер (и попробует установить при отсутствии);
   - определит версию браузера, подберет и скачает подходящий `YandexDriver` в `driver/yandexdriver.exe`;
   - создаст ярлык `KonturAPI` на рабочем столе для запуска `main.pyw`.

## Ручной запуск через uv

```bash
uv python install 3.12
uv sync --python 3.12
uv run --python 3.12 main.pyw
```

## Разработка

```bash
uv sync --python 3.12 --group dev
uv run flake8 .
uv run pytest -q
```

## Синхронизация истории заказов

- История заказов синхронизируется через GitHub автоматически.
- Для хранения используется отдельная ветка `orders-history` в том же репозитории.
- Локальный технический кеш синхронизации хранится в `.history_sync_cache/`.
- При конфликте данных записи объединяются по `document_id` с приоритетом более свежих изменений, чтобы не терять историю.

Переменные окружения:

- `HISTORY_SYNC_ENABLED=0` — полностью выключить синхронизацию.
- `HISTORY_SYNC_BRANCH=<branch_name>` — использовать другую ветку вместо `orders-history`.
- `HISTORY_SYNC_TOKEN=<github_pat>` — токен GitHub для синхронизации без интерактивной авторизации.
- `HISTORY_SYNC_USERNAME=<username>` — логин для basic-auth (по умолчанию `x-access-token`).
- `GITHUB_TOKEN=<github_pat>` — общий токен для автообновления и синхронизации (fallback).
- `GITHUB_USERNAME=<username>` — логин для автообновления (fallback).
